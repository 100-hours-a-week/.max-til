## 🚀 Today I Learned

## | 운영체제

- 운영체제는 컴퓨터의 하드웨어와 사용자가 실행하는 프로그램 사이에서 자원을 관리해주는 시스템 소프트웨어다.

- CPU, 메모리, 디스크, 파일, 네트워크, 입출력 장치 같은 자원을 효율적으로 사용할 수 있도록 관리한다.

- 대표적으로 Windows, macOS, Linux, Android, iOS 등이 운영체제에 해당한다.

### 왜 쓰는가?

- 사용자가 직접 CPU나 메모리 같은 하드웨어를 제어하는 것은 너무 복잡하고 위험하기 때문

- 자원관리와 사용자 편의성, 멀티태스킹, 보안 및 안정성을 이유로 사용한다.


<br>

---

## | 커널

- 커널은 운영체제의 핵심 부분으로, 하드웨어 자원을 직접 관리하는 영역이다.

- CPU 스케줄링, 메모리 관리, 파일 시스템, 프로세스 관리, 입출력 장치 제어 같은 중요한 작업을 담당한다.

- 쉽게 말하면 운영체제 안에서도 가장 중심이 되는 관리자

### 왜 쓰는가?

- 일반 프로그램이 하드웨어에 직접 접근하면 시스템 전체가 불안정해질 수 있다.

- 다라서 커널이 중간에서 하드웨어 접근을 통제하고 필요한 기능만 안전하게 사용할 수 있도록 함


<br>

---

## | 시스템 콜

- 시스템 콜은 사용자 프로그램이 커널에게 기능을 요청하는 통로

- 일반 프로그램은 하드웨어나 커널 자원에 직접 접근할 수 없기 때문에 시스템 콜을 통해 운영체제 기능을 사용한다.

### 왜 쓰는가?

- 모든 프로그램이 마음대로 하드웨어에 접근하면 보안 문제와 시스템 오류가 발생할 수 있기 때문


<br>

## | 가상 메모리

- 가상 메모리는 실제 물리 메모리보다 더 큰 메모리를 사용하는 것처럼 보이게 해주는 메모리 관리 기법

- 각 프로세스는 자신만의 독립적인 메모리 공간을 가진 것처럼 동작

- 실제로는 운영체제가 가상 주소를 물리 메모리 주소로 변환해서 관리함

### 왜 쓰는가?

- 가상 메모리를 사용함으로써 각 프로세스가 독립된 주소 공간을 가지게 되면서 안정성과 보안성이 높아지기 때문

- 또한 자주 사용하지 않는 메모리 영역을 디스크로 옮겨두어 실제 RAM보다 더 큰 메모리를 사용하는 것처럼 처리할 수 있어서


<br>

## | 페이징

- 페이징은 가상 메모리와 물리 메모리를 일정한 크기의 블록으로 나누어 관리하는 메모리 관리 기법

- 가상 메모리의 블록을 페이지라고 하고, 물리 메모리의 블록을 프레임이라고 함

- 고정된 크기의 페이지를 사용하기 때문에 내부 단편화가 발생할 수 있음


### 왜 쓰는가?

-  연속된 큰 메모리 공간을 확보하기 어려운 문제를 해결하기 위해 사용한다.

- 프로그램 전체를 연속된 공간에 올리지 않아도, 페이지 단위로 나누어 여러 위치에 배치할 수 있따


<br>

## | 세그멘테이션

- 세그멘테이션은 메모리를 의미 있는 단위로 나누어 관리하는 메모리 관리 기법

- 코드 영역, 데이터 영역, 스택 영역, 힙 영역처럼 논리적인 단위로 메모리를 나눈다

- 가변적으로 크기가 변하기 때문에 내부 단편화가 생기진 않지만 외부 단편화가 발생한다


### 왜 쓰는가?

- 논리적인 구조를 반영해서 메모리를 관리하기 위해 사용
