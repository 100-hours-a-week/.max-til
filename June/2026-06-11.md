## | 데드락

- 데드락은 여러 프로세스나 스레드가 서로가 가진 자원을 기다리면서 아무 작업도 진행하지 못하는 상태를 의미한다.

- 한국어로는 교착상태라고 부른다.

- 예를 들어 A 스레드는 1번 자원을 가지고 2번 자원을 기다리고, B 스레드는 2번 자원을 가지고 1번 자원을 기다리면 서로 영원히 대기하게 된다.

### 왜 문제가 되는가?

- 데드락이 발생하면 해당 작업들이 더 이상 진행되지 않기 때문이다.

- 서버에서는 요청이 끝나지 않고 계속 대기하면서 응답 지연이나 장애로 이어질 수 있다.

- 특히 여러 락이나 여러 공유자원을 동시에 사용하는 코드에서는 데드락 가능성을 항상 생각해야 한다.

## | 교착상태 발생 조건

- 교착상태가 발생하려면 일반적으로 4가지 조건이 동시에 만족되어야 한다.

- 이 조건들은 상호 배제, 점유와 대기, 비선점, 순환 대기이다.

- 네 가지 조건 중 하나라도 깨지면 교착상태를 예방할 수 있다.

### 1. 상호 배제

- 하나의 자원은 한 번에 하나의 프로세스나 스레드만 사용할 수 있는 상태를 의미한다.

- 하나의 파일, 하나의 프린터, 하나의 락은 동시에 여러 작업이 사용할 수 없음

### 2. 점유와 대기

- 어떤 스레드가 이미 하나의 자원을 가지고 있는 상태에서 다른 자원을 추가로 기다리는 상태를 의미

-  A 스레드가 1번 락을 가진 상태에서 2번 락을 기다리는 경우

### 3. 비선점

- 이미 다른 스레드가 가지고 있는 자원을 강제로 빼앗을 수 없는 상태를 의미한다.

- 즉, 자원을 가진 스레드가 직접 반납할 때까지 기다려야 한다.

### 4. 순환 대기

- 여러 스레드가 서로의 자원을 원형으로 기다리는 상태를 의미한다.

- 예를 들어 A는 B가 가진 자원을 기다리고, B는 C가 가진 자원을 기다리고, C는 다시 A가 가진 자원을 기다리는 구조이다.

### 왜 알아야 하는가?

- 교착상태는 단순히 락을 사용한다고 무조건 발생하는 것이 아니라, 특정 조건들이 동시에 만족될 때 발생한다.

- 따라서 교착상태 발생 조건을 이해하면 락 획득 순서를 통일하거나, 타임아웃을 설정하거나, 필요한 자원을 한 번에 확보하는 방식으로 문제를 예방할 수 있다.

## | 세마포어

- 세마포어는 동시에 접근할 수 있는 작업의 개수를 제한하는 동기화 도구이다.

- 내부적으로 허가권의 개수를 가지고 있으며, 스레드가 자원을 사용하면 허가권을 하나 가져가고 작업이 끝나면 다시 반납한다.

- 허가권이 남아 있으면 접근할 수 있고, 허가권이 없으면 다른 스레드가 반납할 때까지 기다려야 한다.

- 즉, 락과는 다른 개념이다.

### 왜 쓰는가?

- 특정 작업이나 자원에 너무 많은 스레드가 동시에 접근하는 것을 막기 위해 사용한다.

- 예를 들어 DB 커넥션이 10개라면 동시에 10개의 요청까지만 커넥션을 사용하도록 제한할 수 있다.

- 즉, 세마포어는 데이터 하나의 정합성을 직접 지키기보다는 동시에 접근 가능한 개수를 제어하는 데 사용한다.

## | 이진 세마포어

- 이진 세마포어는 허가권의 개수가 0 또는 1만 존재하는 세마포어이다.

- 허가권이 1개뿐이기 때문에 한 번에 하나의 스레드만 자원에 접근할 수 있다.

- 그래서 동작 방식만 보면 뮤텍스나 락과 비슷하게 사용할 수 있다.

### 왜 쓰는가?

- 공유자원에 한 번에 하나의 스레드만 접근하도록 제한하고 싶을 때 사용할 수 있다.

- 예를 들어 하나의 파일에 동시에 여러 스레드가 쓰기 작업을 하면 데이터가 꼬일 수 있기 때문에 이진 세마포어로 접근을 제어할 수 있다.

- 뮤텍스는 보통 락을 획득한 스레드가 직접 해제해야 한다는 소유권 개념이 강하고, 세마포어는 허가권 개수 제어에 초점이 있다는 차이가 있다.

카운팅 세마포어

- 카운팅 세마포어는 허가권의 개수가 여러 개인 세마포어이다.

- 예를 들어 허가권이 5개라면 동시에 최대 5개의 스레드가 접근할 수 있다.

- 허가권을 모두 사용 중이면 6번째 스레드는 누군가 허가권을 반납할 때까지 대기한다.

### 왜 쓰는가?

- 동시에 여러 개까지는 사용해도 되는 자원을 제어하기 위해 사용한다.

- 예를 들어 DB 커넥션 풀, 다운로드 슬롯, 업로드 처리 개수, 작업 큐 처리 개수 제한 등에 사용할 수 있다.

- 카운팅 세마포어는 같은 데이터 하나를 여러 스레드가 동시에 수정해도 된다는 의미가 아니라, 독립적으로 사용할 수 있는 자원이 여러 개 있을 때 접근 개수를 제한하는 개념이다.
