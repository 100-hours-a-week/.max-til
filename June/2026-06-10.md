## 🚀 Today I Learned

## | 동시성

- 동시성은 하나의 CPU에서 여러 작업, 즉 프로세스나 스레드가 동시에 실행되는 것처럼 보이게 하는 개념이다.

- 실제로는 CPU가 여러 작업을 아주 빠르게 번갈아 실행하면서 사용자 입장에서는 동시에 처리되는 것처럼 느껴진다.
### 왜 쓰는가?

- CPU가 하나의 작업만 끝날 때까지 기다리면 시스템 자원이 비효율적으로 사용되기 때문이다.

- 어떤 작업이 파일 입출력이나 네트워크 응답을 기다리는 동안 CPU가 다른 작업을 처리하면 전체 시스템 효율이 좋아진다.
<br>

## | 스케줄러

- 스케줄러는 운영체제의 핵심 구성 요소로, CPU와 같은 시스템 자원을 어떤 프로세스나 스레드에게 줄지 결정하는 역할을 한다.

- 현재 실행 가능한 작업들 중에서 다음에 어떤 작업을 실행할지 선택하는 관리자

- 스케줄러는 작업의 우선순위, 대기 시간, CPU 사용 시간 등을 고려하여 실행 순서를 조정
### 왜 쓰는가?

- 여러 프로세스와 스레드가 동시에 CPU를 사용하려고 하기 때문에 이를 조율하는 기준이 필요하다.

- 스케줄러가 없다면 하나의 작업이 CPU를 오래 독점하거나, 어떤 작업은 계속 실행되지 못하는 기아 상태가 발생할 수 있다.

- 스케줄러는 CPU 이용률을 높이고, 응답 시간을 줄이며, 시스템 전체가 공정하게 만들기 위해 사용
<br>

## | 스케줄링

- 스케줄링은 CPU와 같은 자원을 프로세스와 스레드들이 사용할 수 있도록 실행 순서를 정하는 과정이다.

- 쉽게 말하면 여러 작업을 줄 세우고, 어떤 작업을 언제 얼마나 실행할지 결정하는 운영체제의 정책이다.

### 왜 쓰는가?

- CPU는 한정된 자원이기 때문에 모든 작업을 무작정 동시에 실행할 수 없기 때문


<br>

---

## | 병렬성

- 병렬성은 여러 CPU 코어 또는 다중 프로세서를 사용하여 여러 작업을 물리적으로 동시에 실행하는 개념이다.

- 동시성이 “동시에 실행되는 것처럼 보이게 하는 것”이라면, 병렬성은 “실제로 동시에 실행하는 것”이다

- 따라서 하드웨어에 의존한다.

### 왜 쓰는가?

- 큰 작업을 여러개로 나누어 동시에 처리할 수 있기 때문에
