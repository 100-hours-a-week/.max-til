## 🚀 Today I Learned

## | ORM

- ORM은 `Object Relational Mapping`의 약자로, 객체와 관계형 데이터베이스 테이블을 매핑해주는 기술이다.

- 자바에서는 객체를 중심으로 개발하지만, 데이터베이스는 테이블, 행, 열 구조로 데이터를 저장한다.

- ORM은 이 둘 사이의 구조 차이를 줄여주고, 객체를 저장하면 테이블에 데이터가 저장되도록 도와준다.

### 왜 쓰는가?

- 자바 개발자는 객체를 중심으로 비즈니스 로직을 작성하는데, 데이터베이스는 테이블 중심으로 동작하기 때문이다.

- ORM을 사용하면 객체와 테이블 간 매핑을 프레임워크가 처리해주기 때문에 반복적인 SQL 매핑 코드가 줄어든다.

## | JPA

- JPA는 `Java Persistence API`의 약자로, 자바에서 ORM을 사용하기 위한 표준 인터페이스이다.

- JPA 자체가 실제로 동작하는 구현체는 아님 , 실제 동작은 Hibernate 같은 구현체가 담당

- 개발자는 JPA 표준 방식으로 코드를 작성하고, Hibernate가 내부에서 SQL 생성과 DB 연동을 처리한다.

### 왜 쓰는가?

- 특정 ORM 구현체에 강하게 의존하지 않고, 표준 방식으로 객체와 데이터베이스를 매핑하기 위해서이다.

- JPA를 사용하면 Entity를 중심으로 데이터를 저장하고 조회할 수 있다

<br>

## | Entity

- Entity는 데이터베이스 테이블과 매핑되는 자바 클래스이다.

- 보통 하나의 Entity 클래스는 하나의 테이블과 연결된다.

### 왜 쓰는가?

- 데이터베이스 데이터를 자바 객체로 다루기 위해서이다.

- SQL 결과를 단순한 값 묶음으로 보는 것이 아니라, 의미 있는 도메인 객체로 다루면 비즈니스 로직을 더 객체지향적으로 작성할 수 있다.

- 다만 Entity는 핵심 도메인 객체이기 때문에 요청/응답에 그대로 사용하기보다는 DTO와 분리하는 것이 좋음!

<br>

## | 영속성 컨텍스트

- 영속성 컨텍스트는 JPA가 Entity를 관리하는 메모리 공간이다.

- Entity를 조회하거나 저장하면 JPA는 해당 Entity를 영속성 컨텍스트 안에서 관리한다.

- 같은 트랜잭션 안에서 동일한 Entity를 다시 조회하면 DB를 다시 조회하지 않고 영속성 컨텍스트의 1차 캐시에서 가져올 수 있다.

- 또한 Entity의 변경 사항을 추적하다가 트랜잭션이 끝날 때 자동으로 DB에 반영할 수 있다

### 왜 쓰는가?

- Entity 객체를 단순히 DB 데이터로만 보는 것이 아니라, 트랜잭션 안에서 상태를 가진 객체로 관리하기 위해서

- JPA가 언제 데이터를 저장, 변경, 조회하는지 정확히 이해하고, 예측 가능한 데이터 처리와 오류 없는 트랜잭션 관리를 하기 위해서

<br>

## | 연관관계 매핑

- 연관관계 매핑은 Entity 객체 간의 관계를 데이터베이스 외래 키와 연결하는 방식이다.

- 데이터베이스에서는 외래 키로 테이블 간 관계를 표현하고, 자바 객체에서는 참조 필드로 객체 간 관계를 표현한다.


### 왜 쓰는가?

- 객체지향적으로 연관된 데이터를 탐색하기 위해서이다.

- SQL 중심으로 개발하면 게시글의 작성자를 조회하기 위해 직접 JOIN SQL을 작성해야 한다.

- 다만 잘못 설계하면 N+1 문제나 복잡한 의존 관계가 생길 수 있음
<br>

## | Proxy

- Proxy는 JPA에서 실제 Entity 대신 먼저 만들어지는 가짜 객체이다.

- 주로 지연 로딩에서 사용된다

- 연관된 Entity를 처음부터 조회하지 않고, 실제로 필요할 때 DB에서 조회하기 위해 사용한다.

### 왜 쓰는가?

- 필요하지 않은 연관 데이터를 처음부터 모두 조회하면 성능이 떨어질 수 있기 때문이다.

- 게시글 목록을 조회하는데 작성자, 댓글, 좋아요, 첨부파일을 모두 즉시 조회하면 불필요한 JOIN과 데이터 조회가 발생한다.

- Proxy를 사용하면 실제로 필요한 순간까지 조회를 미룰 수 있어 성능 최적화에 도움이 된다.

- 다만 Proxy 객체를 반복문 안에서 계속 접근하면 지연 로딩이 여러 번 발생해 N+1 문제가 생길 수 있으므로 주의해야 함

## 💭 오늘의 회고

오늘은 ORM, JPA, Entity, 영속성 컨텍스트, 연관관계 매핑, Proxy에 대해 정리해보았다. 처음에는 JPA를 단순히 SQL을 대신 만들어주는 기술 정도로 생각했지만, 정리하면서 JPA의 핵심은 객체와 데이터베이스 사이의 차이를 줄이고, 엔티티를 영속성 컨텍스트 안에서 관리하는 것이라는 점을 알게 되었다.

특히 ORM은 객체지향 코드와 관계형 데이터베이스 사이의 구조 차이를 해결하기 위해 등장한 기술이라는 점이 중요했다. 자바에서는 객체를 중심으로 개발하지만, 데이터베이스는 테이블 중심으로 데이터를 저장한다.

JPA를 공부하면서 가장 핵심이라고 느낀 부분은 영속성 컨텍스트였다. JPA는 단순히 `save`, `find` 같은 메서드만 제공하는 것이 아니라, 엔티티를 메모리 공간에서 관리하면서 1차 캐시, 변경 감지, 쓰기 지연 같은 기능을 제공한다. 그래서 영속 상태의 엔티티는 값을 변경하기만 해도 트랜잭션이 끝날 때 자동으로 DB에 반영될 수 있다.

결국 JPA는 SQL을 몰라도 되는 기술이 아니라, SQL과 객체지향 사이를 더 효율적으로 연결해주는 기술이라고 느꼈다 앞으로 JPA를 사용할 때는 단순히 어노테이션을 붙이는 데서 끝내지 않고, 이 코드가 어떤 SQL로 실행될지, 영속성 컨텍스트 안에서 엔티티가 어떤 상태인지까지 생각하면서 개발해야겠다고 느꼈다.
