## 🚀 Today I Learned

## | Git

- `Git`은 로컬 환경에서 파일의 변경 이력을 관리하는 **분산 버전 관리 시스템**이다.

- Git을 사용하면 파일을 수정한 기록을 남길 수 있고,  
  이전 상태로 되돌리거나 변경 내용을 비교할 수 있다.

- Git은 내 컴퓨터 안에서 동작하기 때문에,  
  인터넷이 없어도 커밋 기록을 만들고 관리할 수 있다.

### 왜 쓰는가?

- 개발을 하다 보면 코드를 수정하는 과정에서 오류가 발생하거나,  
  이전 코드로 되돌아가야 하는 상황이 생긴다.

- Git을 사용하면 단순히 파일을 복사해서 백업하는 것이 아니라,  
  어떤 파일이 언제, 어떤 내용으로 변경되었는지 기록할 수 있다.

- 또한 커밋 단위로 작업 내용을 저장하기 때문에  
  문제가 생겼을 때 특정 시점으로 되돌아가거나, 변경 내용을 비교하면서 원인을 찾을 수 있다.

- Git은 `분산 버전 관리 시스템`이기 때문에  
  중앙 서버에만 의존하지 않고 각 개발자의 컴퓨터에도 전체 저장소 기록이 저장된다.

## | GitHub

- `GitHub`는 Git 저장소를 온라인에서 관리하고 공유할 수 있게 해주는 서비스이다.

- Git이 내 컴퓨터에서 변경 이력을 관리하는 도구라면,  
  GitHub는 그 저장소를 웹에서 확인하고 다른 사람과 협업할 수 있게 해준다.

- GitHub를 사용하면 코드 백업, 협업, Pull Request, Issue 관리 등을 할 수 있다.

### 왜 쓰는가?

- Git은 로컬 컴퓨터에서도 버전 관리를 할 수 있지만,  
  내 컴퓨터 안에만 저장되어 있으면 다른 사람과 코드를 공유하기 어렵다.

- GitHub를 사용하면 Git 저장소를 온라인 원격 저장소에 올릴 수 있다.

- 이를 통해 다른 컴퓨터에서도 같은 프로젝트를 가져와 작업할 수 있고,  
  팀원들과 코드를 공유하고 변경 사항을 합칠 수 있다.

- 또한 GitHub는 Pull Request, Issue, 코드 리뷰 기능을 제공하기 때문에  
  단순한 코드 저장소를 넘어서 협업을 체계적으로 관리하는 공간으로 사용할 수 있다.

## | 클래스 상속 관계

- `상속`은 기존 클래스의 속성과 기능을 새로운 클래스가 물려받아 사용하는 객체지향 프로그래밍의 개념이다.

- 상속을 사용하면 여러 클래스에서 공통으로 사용하는 코드를 부모 클래스에 작성하고,  
  자식 클래스는 필요한 기능만 추가하거나 수정할 수 있다.

- 부모 클래스는 공통된 필드와 메서드를 가지고,  
  자식 클래스는 부모 클래스의 기능을 물려받으면서 자신만의 특징을 추가할 수 있다.

### 왜 쓰는가?

- 여러 클래스에서 같은 필드와 메서드를 반복해서 작성하면 코드가 길어지고 관리하기 어려워진다.

- 상속을 사용하면 공통된 부분은 부모 클래스에 한 번만 작성하고,  
  자식 클래스에서는 필요한 특징만 추가하면 된다.

- 나중에 공통 기능을 수정해야 할 때 부모 클래스만 수정하면 되기 때문에 유지보수가 쉬워진다.

## 💭 오늘의 회고

오늘은 Git과 GitHub의 기본 흐름뿐만 아니라, 브랜치를 나누어 작업하고 다시 병합하는 과정에서 충돌이 왜 발생하는지 직접 경험했다.

처음에는 `main`, `sub1`, `sub2` 브랜치가 각각 어떤 상태인지 헷갈렸지만, 브랜치가 만들어진 시점과 각 브랜치에서 수정한 파일을 따라가면서 충돌의 원인을 이해할 수 있었다.

특히 충돌은 단순한 오류가 아니라, 서로 다른 브랜치의 변경 내용 중 어떤 것을 최종 결과로 남길지 사람이 직접 판단해야 하는 상황이라는 점이 기억에 남았다.

이번 충돌 해결은 GUI 도구의 버튼을 사용한 것이 아니라, 충돌이 발생한 파일을 직접 열고 `<<<<<<< HEAD`, `=======`, `>>>>>>>` 표시를 확인한 뒤 필요 없는 부분을 지우고 원하는 내용만 남기는 방식으로 진행했다.

이 과정을 통해 Git이 충돌 상황을 파일 안에 어떻게 표시하는지 알 수 있었고, 충돌 해결은 결국 최종적으로 남길 내용을 직접 정리하는 과정이라는 것을 이해했다.

다만 아직은 충돌 표시를 직접 지워서 해결하는 흐름만 경험했기 때문에, 다음에는 터미널 명령어나 Git의 추가 기능을 활용해서 충돌을 더 체계적으로 해결하는 방법도 공부해봐야겠다고 느꼈다.
